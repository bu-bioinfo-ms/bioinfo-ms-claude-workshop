# Module 0 — Getting started

**Goal:** understand what this workshop *is* and how you engage with it, get set
up, and meet your guide.

**Time:** ~15 min · **Checkpoint:** `PROGRESS.md` has your name and model, and
you've had your first exchange with the guide.

---

## The idea

This repository is a **self-paced bootcamp that Claude runs for you.** You don't
read a PDF and follow along — you open the repo in Claude Code, and Claude acts
as your guide: it figures out where you are, teaches the next module, has you do
the hands-on part, checks your work, and records your progress back into the
repo. Next time you open it, it picks up where you left off.

You're learning agentic coding workflows *by using one*. Every mechanism of this
workshop is also one of its lessons:

| The workshop uses… | …which teaches you |
|---|---|
| A `PROGRESS.md` it reads & updates | Making decisions durable in markdown |
| Claude guiding you | Driving an agent; plan vs. auto mode |
| A growing `CLAUDE.md` | Project memory & how instructions shape behavior |
| One commit per module | Conventional commit strategy |
| A `workshop` skill | What skills are and when to reach for them |

## Setup

1. **Fork** this repo to your own GitHub account (all your work lives in your
   fork).
2. **Clone** your fork and open it in Claude Code.
3. **Pick a model.** For teaching and reasoning-heavy guidance, Opus is a good
   default; you'll learn *how to choose* deliberately in Module 2. (Just notice
   for now that model choice is a lever you control.)

## Activity — meet your guide

1. Start a Claude Code session in your fork. Because `CLAUDE.md` tells the agent
   it's a workshop guide, it should greet you and check `PROGRESS.md`.
2. Tell it your name. It will record it in `PROGRESS.md`.
3. Ask it: **"What is this workshop and how does it work?"** and **"Where am I?"**
   Confirm it can explain the loop and read your progress.

## Checkpoint ✅

- `PROGRESS.md` shows your name and chosen model.
- Module 0 is marked `✅ done` and Module 1 is unlocked.
- You can state, in one sentence, how you'll resume next time. (Answer: open the
  repo and say *"let's continue"* — the guide reads `PROGRESS.md`.)
